# Chain Verification Report

> **Status:** PENDING — run `pnpm chain:verify` against a live node to populate this file.

This file is generated by `scripts/chain-verify.ts`. It must be updated and committed before any
on-chain integration code is merged (see CLAUDE.md section 3).

## What to verify

1. ink! contract version compatibility (upload a hello-world, confirm it instantiates)
2. Whether chain extensions exist (expect: no — compose via frontend batchAll instead)
3. Identity pallet field set (`api.tx.identity.setIdentity` parameter types)
4. Proxy types available (`api.tx.proxy.addProxy` — confirm `Any`, `NonTransfer`, `Governance`, `Staking`, `IdentityJudgement`, `CancelProxy`)
5. Multisig `DepositBase` and `DepositFactor` constants
6. Bounty `BountyDepositBase` and `CuratorDepositMin` constants
7. Whether `utility.batchAll` is available (expected: yes)
8. `balances.existentialDeposit`

## Running

```bash
# Against local dev node (start it first):
./portaldot --dev --tmp --rpc-port 9944 --rpc-cors all

pnpm chain:verify

# Against mainnet:
pnpm chain:verify --mainnet
```
